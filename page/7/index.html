缓存穿透

发表于 2020-01-09 | 分类于 技术

缓存穿透

现象

正常情况下，我们去查询数据都是存在。那么请求去查询一条压根儿数据库中根本就不存在的数据，也就是缓存和数据库都查询不到这条数据，但是请求每次都会打到数据库上面去。这种查询不存在数据的现象我们称为缓存穿透。试想一下，如果有黑客会对你的系统进行攻击，拿一个不存在的id去查询数据，会产生大量的请求到数据库去查询。可能会导致你的数据库由于压力过大而宕掉。

方案

  1. 缓存空值：之所以会发生穿透，就是因为缓存中没有存储这些空数据的key。从而导致每次查询都到数据库去了。那么我们就可以为这些key对应的值设置为null 丢到缓存里面去。后面再出现查询这个key的请求的时候，直接返回null。这样，就不用在到数据库中去走一圈了，但是别忘了设置过期时间。
  2. 布隆过滤器：BloomFilter 类似于一个hbase set用来判断某个元素（key）是否存在于某个集合中。这种方式在大数据场景应用比较多，比如Hbase中使用它去判断数据是否在磁盘上。还有在爬虫场景判断url是否已经被爬取过。这种方案可以加在第一种方案中，在缓存之前在加一层BloomFilter，在查询的时候先 BloomFilter 去查询key是否存在，如果不存在就直接返回，存在再走查缓存->查DB。

针对于一些恶意攻击，攻击带过来的大量key是不存在的，那么我们采用第一种方案就会缓存大量不存在key的数据。此时我们采用第一种方案就不合适了，我们完全可以先对使用第二种方案进行过滤掉这些key。针对这种key异常多、请求重复率比较低的数据，我们就没有必要进行缓存，使用第二种方案直接过滤掉。而对于空数据的key有限的，重复率比较高的，我们则可以采用第一种方式进行缓存。

缓存击穿

现象

缓存击穿是我们可能遇到的第二个使用缓存方案可能遇到的问题。在平常高并发的系统中，大量的请求同时查询一个key时，此时这个key正好失效了，就会导致大量的请求都打到数据库上面去。这种现象我们称为缓存击穿。会造成某一时刻数据库请求量过大，压力剧增。

方案

  1. 对热点key，加互斥锁
  2. 设置热点数据永不过期
  3. 资源保护，服务降级

缓存雪崩

现象

缓存雪崩，是指缓存中设置了大批量相同过期时间的数据同时过期失效，而在这一刻访问量剧增，缓存近乎失效，所有请求全部转向DB，DB瞬时压力过重雪崩，甚至down机。和缓存击穿不同的是，缓存击穿指并发查同一条数据，缓存雪崩是不同数据都过期了，很多数据都查不到从而查数据库

方案

  1. 事前：使用集群缓存，保证缓存服务的高可用。这种方案就是在发生雪崩前对缓存集群实现高可用，如果是使用Redis，可以使用主从+哨兵，Redis Cluster 来避免Redis 全盘崩溃的情况。过期时间设置随机，防止同一时间大量数据过期现象发生，为了避免这些热点的数据集中失效，那么我们在设置缓存过期时间的时候，我们让他们失效的时间错开。比如在一个基础的时间上加上或者减去一个范围内的随机值。

  2. 事中：ehcache本地缓存+Hystrix限流&降级，避免MySQL被打死使用ehcache本地缓存的目的也是考虑在Redis Cluster 完全不可用的时候，ehcache本地缓存还能够支撑一阵。使用Hystrix进行限流&降级，比如一秒来了5000个请求，我们可以设置假设只能有一秒2000个请求能通过这个组件，那么其他剩余的3000请求就会然后去调用我们自己开发的降级组件（降级），比如设置的一些默认值呀之类的。以此来保护最后 MySQL不会被大量的请求给打死。

  3. 事后：开启Redis持久化机制，尽快恢复缓存集群一旦重启，就能从磁盘上自动加载数据恢复内存中的数据。

基金理财技巧 - 指数基金

发表于 2020-01-05 | 分类于 投资理财

深交所指数

深证成指 （399001） - 代表深证中小盘

深证100（399330）- 代表深证大盘

上交所指数

上证50 （000016）- 代表大盘指数（约等于买金融股 + 两桶油【中石油，中石化】）

上证180（000010）- 代表大盘指数

中证指数

深证和上证合作，成立中证指数。

沪深300 （000300）- 相当于深证 100 + 上证180。也代表了大盘指数

中证500 - 代表小盘。正好和沪深 300 形成风险对冲，如果两个都买了，那就相当于买了全中国最优秀的股票。

指数基金的挑选

主要看跟踪误差，跟踪误差越小越好。日误差 <= 0.5%, 年误差 <= 1.5%。最好不买增强型指数基金。

指数基金管理费一般只有 0.5% 左右。增强型指数基金稍高，大约 1% 左右。且指数基金不需要经常调仓，换手率应该选在 50% 以内的。

如果选择不是被动形的指数基金，那么还不如买主动型的股票型基金。

基金理财技巧 - 股票型基金

发表于 2020-01-05 | 分类于 投资理财

相关概念

  1. 股票基金的风险是所以基金里面最高的
  2. 三大基金评级机构，晨星，理柏，惠誉
  3. 主动型股票基金存在老鼠仓等利益问题

主动形股票基金

  1. 看基金公司，大的公司更好
  2. 基金行业第一梯队：天弘，共银瑞信，易方达，博时，招商，南方，嘉实，华夏，建信，中银，汇添富，鹏华，广发，富国，银华，华安，华宝，兴全，兴业，上投摩根，大成。（深色的更好些）。挑选股票型基金的时候优先选这些公司的。
  3. 看基金经理。最好 35 岁以上，管理这只基金的时间越长越好。
  4. 基金经理投资风格。尽量选哪些长期持有，或者操作不是那么激进的基金经理。换手率半年会更新一次，换手率在 20% - 200% 之间属于正常范围，一旦接近或者超过 500%，那么就不要考虑了。

  1. 查看晨星投资风格箱。选择顺序为。小盘价值型，大盘价值型，大盘成长型，小盘成长型。（收益从高到低）
  2. 基金成立时间比较久的优先，至少超过 3年，最好 5 年以上，越长越好
  3. 长时间业绩稳定（同类排名 20% 以内最好，且波动小）

  1. β系数。在 0.5 和 1.5 之间波动。β系数主要有两种选择，高 β 策略 - 适合激进型投资者和牛市。低 β 策略，适合稳健型投资者和熊市。
  2. α系数。α系数越高越好，越能表示基金经理水平高
  3. R平方。代表 β系数和 α系数的准确度，越高越好
  4. 夏普比率。夏普比率越高，说明基金性价比越高，越值得拥有
  5. 如果不上用上面的各种系数，那么只看晨星网的星数就好。
  6. 不要选规模较大的基金。如果是投资中小盘的基金，那么规模在 10 -20 亿比较合适。如果投资的是大盘股的基金，那么选择 30 -100 亿之前比较合适。超过 100 亿低于 2亿的就不要买了。
  7. 股票型基金的管理费比债券型和货币型都要高，一般在 1.5% 左右。申购费大约 0.15% 左右，赎回费 0.5%。优先考虑小于或者等于这三个收费标准的。

基金理财技巧 - 混合型基金

  1. 混合型基金比债券型的风险和收益都要高。混合型基金包含股票和债券。
  2. 股票型基金的股票比例为至少 80%
  3. 混合型基金最大的特点是风险对冲。
  4. 混合型基金的优点就是它的缺点（太灵活，比例不一，比较考验基金经理）

基金理财技巧 - 纯债基金

发表于 2020-01-04 | 分类于 投资理财
  1. 历史数据表明，纯债基金都是保本的（持有时间至少一年）
  2. 纯债基金平均年化在 6% 左右，优秀的 8% 左右。
  3. 激进型投资者可将 10% 的资金投资纯债基金。保守型可将 20%-40% 的资金投资纯债基金。
  4. 因为债券的波动小，所以不推荐债券型指数基金（指数基金还是推荐股票指数基金，不推荐债券型指数基金）
  5. 债基中后面的 ABC 的含义

如果是长期持有，比如一年，那么就选择一次性收取收购赎回的比较划算。所以推荐后端收费模式，长期持有，收益稳定，且费率低。

选取排名前 30 的基金：

  1. 依次点击，排除持仓有可转债的。
  2. 排除小公司的基金
  3. 排除基金规模超过 20 亿的
  4. 只选取同一家基金公司的排名符合上述的第一位基金，其余同一家公司的基金剔除，只选出这家基金公司的的王牌纯债紧急。
  5. 按照以上规则筛选出10只基金。
  6. 找出综合费率最低的 5 只
  7. 再次在这五只中寻找出基金成立时间较长的，基金经理年纪较大的，管理这只时间较长的的 3 只。
  8. 以上选出的三只就能作为个人的纯债配置。

为了方便基金经理投资，防止例如突然发生的大额赎回的情况影响运作，定期开放式债券基金比纯债基金有更大的收益，但是牺牲了灵活性。挑选定开债的技巧如下：

  1. 还是只能选择纯债基金
  2. 别选择杠杆率太高的基金，最好不好超过 160% （杠杆率：例如某只定开债价值 10 亿，杠杆率为 200%，那么就可以向银行贷款 20 亿。只要这 20 亿的收益大于银行利息，那么就是赚的，但是杠杆率是把双刃剑，有利有坏，所以不要选择太高的）
  3. 尽量选择定期为1年的定开债，半年太短，2-3年太长。选择好符合的基金后记住下次开放购买时间，到时候买入就行。

为什么不直接买债券，而选择买债券基金？

  1. 债券品种较少，投资门槛高
  2. 债券流动性差
  3. 债券投资性价比低
  4. 债券投资复杂

基金分红，净值：

假如购买了某基金 1万份，每份1元，总价值 1万元。一年后，基金赚钱了，每份变成了2元，这个时候你的总收益就是 2万元。假如进行分红，每份基金分红1元，你持有 1万份，所以分红 1 万元。所以分红后你的基金还是 1 万份，不过从原来的每份 2 元变成了每份 1 元。如果进行的是红利再投，那么你持有的基金份额会变成 2 万份，每份 1元。总的来说，基金分红对投资者无任何影响，只是左口袋拿到有口袋的会计游戏。一个基金的好坏不能用是否分红来衡量。衡量基金最关键的指标是基金的累积净值，单位净值是分红后的，累积净值是包含分红的。累计净值/单位净值=基金的增长倍数。

基金理财技巧 - 货币基金

发表于 2020-01-04 | 分类于 投资理财

货基基本知识

  1. 货币基金收益率不高（2%-5%之间），特点是高安全，高稳定。收益大约是银行定期的2倍左右。

  2. 货币基金的重要作用是充当现金流工具。

  3. 应该随时准备 3-6 的紧急备用金（例如一个月 4000，那么就至少准备 12000 的紧急备用金）。如果总资产不够备用金，那么应该全买货币基金，不建议进行其他高级投资。

  4. 货币基金的价格永远是1元，赚的钱以份额方式存在。例如买了1万份基金，价值1万元，第二天涨到了1万零1元，也就是1万零1份。那么多出来的1元就是万份收益。假如一年 365 天，每天的万份收益都是 1 元，那么1年能赚 365 元，那么实际的年收益就是 3.65%。投资货币基金不能看七日年化收益率和每万份收益，因为这两个值都可以在短期时间内快速拉升，总的还是要看长期收益。

  5. 货基的收益很低，不必经常更换，作用不大，收益变动也不大，反而耽误了收益。

  6. 货基后面带有字母 ABC，A 代表起卖价格 100 元起，B 代表 500 万起卖。收益上 A 类比 B 类低 0.2% 左右。这是为了区分个人投资者和机构投资者，方便基金公司管理。但是现在新出的货基都不区分 ABC 类了。建议购买的时候也优先买不分类的。

基金爬虫

发表于 2019-12-31 | 分类于 奇淫技巧

爬取天天基金网指定基金数据，计算出平均值。可发送到指定邮箱，结合 linux crontab 更好。

# -*- coding: utf-8 -*-

import datetime
import smtplib
from email.header import Header
from email.mime.text import MIMEText

import requests
from bs4 import BeautifulSoup
import re
import numpy as np
import pandas as pd


def get_url(url, params=None, proxies=None):
    """
    请求页面信息
    :param url:
    :param params:
    :param proxies:
    :return:
    """
    rsp = requests.get(url, params=params, proxies=proxies)
    rsp.raise_for_status()
    return rsp.text


def get_fund_data(code, per=10, sdate='', edate='', proxies=None):
    """
    抓取数据
    :param code:
    :param per:
    :param sdate:
    :param edate:
    :param proxies:
    :return:
    """
    url = 'http://fund.eastmoney.com/f10/F10DataApi.aspx'
    params = {'type': 'lsjz', 'code': code, 'page': 1, 'per': per, 'sdate': sdate, 'edate': edate}
    html = get_url(url, params, proxies)
    soup = BeautifulSoup(html, 'html.parser')

    pattern = re.compile(r'pages:(.*),')
    result = re.search(pattern, html).group(1)
    pages = int(result)

    heads = []
    for head in soup.findAll("th"):
        heads.append(head.contents[0])
    records = []
    page = 1
    while page <= pages:
        params = {'type': 'lsjz', 'code': code, 'page': page, 'per': per, 'sdate': sdate, 'edate': edate}
        html = get_url(url, params, proxies)
        soup = BeautifulSoup(html, 'html.parser')

        for row in soup.findAll("tbody")[0].findAll("tr"):
            row_records = []
            for record in row.findAll('td'):
                val = record.contents

                if val == []:
                    row_records.append(np.nan)
                else:
                    row_records.append(val[0])
            records.append(row_records)
        page = page + 1
    np_records = np.array(records)
    data = pd.DataFrame()
    for col, col_name in enumerate(heads):
        data[col_name] = np_records[:, col]

    return data


def send_email(msg):
    """
    发送邮件
    :param to_send:
    :return:
    """
    # 第三方 SMTP 服务
    mail_host = "smtp.exmail.qq.com"  # 设置服务器
    mail_user = ""  # 用户名
    mail_pass = ""  # 口令

    sender = ''
    receivers = ['']  # 接收邮件，可设置为你的QQ邮箱或者其他邮箱

    message = MIMEText(msg, 'plain', 'utf-8')
    message['From'] = Header("基金每日均值", 'utf-8')
    message['To'] = Header("", 'utf-8')

    subject = '基金每日均值'
    message['Subject'] = Header(subject, 'utf-8')

    try:
        smtpObj = smtplib.SMTP()
        smtpObj.connect(mail_host, 25)  # 25 为 SMTP 端口号
        smtpObj.login(mail_user, mail_pass)
        smtpObj.sendmail(sender, receivers, message.as_string())
        print "邮件发送成功"
    except Exception as e:
        print "Error: 无法发送邮件：" + str(e)
