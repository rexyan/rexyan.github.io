关于

  • 昵称：星尘【出自：”明月清风晓星尘，傲雪凌霜颂子琛”】
  • 爱好：看书、纪录片

　　博客没有开通评论，只是想写点自己心里想写的东西。博客地址也没有对任何人提及。所以，这里相当于是自己的一个心情抒发地，管他什么爱恨情仇、公司主管。人的一生很长也很短，能记录一点琐事，以后看看，不妨是件好事。如果你看到了这篇关于本站的文章，那么说明搜索引擎收录了本站。遇见你，很高兴！博客嘛，肯定都是写些和自己相关的看法和意见，每个人的看法和意见不一样，当你在博客中见到与你看法或者意见不同的观点时，还请各位朋友包容、海涵。

　　知识这两个字，我们小时候就开始使用，但是真正有人能把知识表达清楚的却很少，特别是我们这种类似直男且不会说话的人。个人认为，一个知识点，首先是自己心里要先明白，其次应该尝试将其用自己的语言组织表达出来，最后渴望达到大家都能明白，且大家看了得到了收获，并觉得愉悦。正如这篇关于本站的说明就是个失败的例子。

　　写博客一年多了，现在大多数的技术文章都在为知笔记中写了，而是记录在博客里面。工作中有时候也需要打开博客，便于查阅一些东西。博客依然没有开通评论，也没有对外推广交换友链什么的，但是百度和谷歌的收录量分别是 30 多和 300 多。使用 hexo 之后的确比以前自己搭建省事很多，以前不是因为数据丢失就是服务器的原因导致博客都没有继续使用下去。前不久将这个域名又续费了三年，这表明我对这个使用起来还是很满意的。自己使用了不光是 coding 的 git page 也使用了 github 的，但是二者的内容是一模一样的。这也算是一个备份吧。针对于图片也是，自己能不使用就不使用。因为图片不是存储在自己空间的，万一哪一天存储图片的 CDN 停止提供服务了那么将导致内容有所损坏。 — 20181222

　　对 “关于” 页面信息稍微做了些调整，增加了一些字段，比如喜欢的动漫，纪录片等。最近听了一些关于大数据和数据化的讲座，每听完一场都觉得要保护自己的隐私，现在的企业，国家政府部门都在对个人信息进行大量的整合和分析。所以自己一直没有在博客中提到自己的公司，名字等。博客也没有对外，主要是自己记录一些学习的信息，很少时间在这上面发牢骚和写其他的东西(其实应该都多写写书评和读后感)以后在工作中也应该尽量做到少分享博客地址，尽量保护自己的信息。— 20190324
