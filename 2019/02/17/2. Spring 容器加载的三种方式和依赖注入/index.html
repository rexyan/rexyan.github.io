Spring 容器加载的三种方式

  1. 类路径获得配置文件 (最常用)

    ApplicationContext context = new ClassPathXmlApplicationContext("beans.xml");

  2. 文件系统路径获得配置

    ApplicationContext context1 = new FileSystemXmlApplicationContext("/Users/rex/IdeaProjects/day02-spring/src/beans.xml");

  3. 使用BeanFactory(了解)

    String path = "/Users/rex/IdeaProjects/day02-spring/src/beans.xml";
BeanFactory beanFactory = new XmlBeanFactory(new FileSystemResource(path));

    BeanFactory和ApplicationContext的区别

    BeanFactory 采取延迟加载，第一次getBean时才会初始化Bean。而ApplicationContext是即时加载的，是对BeanFactory扩展，提供了更多功能

装配 Bean 的三种方式

所谓装配 Bean 其实就是在 xml 中写一个 bean 标签

方式1

  • Xml 装配
    <!-- 方式1 -->
<bean id="userService1" class="me.yanrs.service.UserServiceImpl" ></bean>
  • 使用
    // 方式1 获取对象
ApplicationContext context = new ClassPathXmlApplicationContext("beans.xml");
UserServiceImpl userService1 = (UserServiceImpl) context.getBean("userService1");
userService1.add();

方式2

  • Xml 装配
    <!-- 方式2 通过静态工厂方法 -->
<bean id="userService2" class="me.yanrs.service.UserServiceFactory" factory-method="CreateUserServiceFactory"> </bean>
  • 创建静态工厂
    `java
    package me.yanrs.service;

public class UserServiceFactory {
// 创建一个静态工厂方法
public static UserServiceImpl CreateUserServiceFactory(){
return new UserServiceImpl();
}
}

- 使用
```java
 // 方式2 通过静态工厂获取对象
ApplicationContext context2 = new ClassPathXmlApplicationContext("beans.xml");

Bean 作用域

  1. singleton(掌握)：在Spring IoC容器中仅存在一个Bean实例，Bean以单例方式存在，默认值

  2. prototype(掌握)：每次从容器中调用Bean时，都返回一个新的实例，即每次调用getBean()时 ，相当于执行new XxxBean()

  3. request(了解): 每次HTTP请求都会创建一个新的Bean，该作用域仅适用于WebApplicationContext环境

  4. session(了解): 同一个HTTP Session 共享一个Bean，不同Session使用不同Bean，仅适用于WebApplicationContext 环境

  5. globalSession(了解): 一般用于Portlet应用环境，该作用域仅适用于WebApplicationContext 环境

  • xml 配置

    <?xml version="1.0" encoding="UTF-8"?>
<beans xmlns="http://www.springframework.org/schema/beans"
     xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
     xsi:schemaLocation="
http://www.springframework.org/schema/beans http://www.springframework.org/schema/beans/spring-beans.xsd">

  <!-- 配置一个 bean（对象） -->
  <bean id="userService" class="me.yanrs.service.UserServiceImpl" scope="prototype"></bean>
</beans>
  • 调用
    `java
    package com.yanrs.test;

    System.out.println(userService);
    System.out.println(userService2);

    当 xml 中 scope="singleton"或者为空时，代表单例模式，输出为：
    me.yanrs.service.UserServiceImpl@1f28c152
    me.yanrs.service.UserServiceImpl@1f28c152

    当 xml 中 scope="prototype"时，代表多例模式，输出为：
    me.yanrs.service.UserServiceImpl@6ddf90b0
    me.yanrs.service.UserServiceImpl@57536d79
     */
}

1.1 构造方法注入

使用前提是类要有相关的构造函数

<?xml version="1.0" encoding="UTF-8"?>
<beans xmlns="http://www.springframework.org/schema/beans"
       xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
       xsi:schemaLocation="
http://www.springframework.org/schema/beans http://www.springframework.org/schema/beans/spring-beans.xsd">

    <!-- 构造方法注入属性值 -->
    <bean id="stu" class="me.yanrs.model.Student">
        <!-- 相当于调用了  public Student(String username, String password) 这个构造方法-->
        <constructor-arg name="username" value="zhangsan"></constructor-arg>
        <constructor-arg name="password" value="123456"></constructor-arg>
    </bean>
</beans>

1.4 集合注入
集合的注入都是给<property>添加子标签
        数组：<array>
        List：<list>
        Set：<set>
        Map：<map> ，map存放k/v 键值对，使用<entry>描述
        Properties：<props>  <prop key=""></prop>  
普通数据：<value>
引用数据：<ref>

新建一个类，里面属性类型为list set map properties 数组等

@Component和@Component(“UserService”)代码示例

  • @Repository @Service @Controller @Autowired…
    web开发中，提供3个@Component注解衍生注解 @Repository @Service @Controller取代 <bean class="">
@Repository(“名称”)：dao层
@Service(“名称”)：service层
@Controller(“名称”)：web层

@Autowired：自动根据类型注入
@Qualifier(“名称”):指定自动注入的id名称

@Resource(name="名称"):等同于 Autowired + Qualifier

@Scope("prototype"):将类设置为多例，默认是单例

@PostConstruct: 相当于 init-method 方法
@PreDestroy: 相当于 destroy-method 方法

web 开发中调用流程为 action 调用 service，service 调用 dao，我们需要在 dao 层加上 Repository 注解，service层加上 Service 注解，web 层加上 Controller 注解，并使用 Autowired 注解自动注入变量的值。也可以使用 Qualifier 注解指定 id 注入。

dao 层

package me.yanrs.action;

import me.yanrs.model.User;
import me.yanrs.service.IUserService;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.beans.factory.annotation.Qualifier;
import org.springframework.stereotype.Controller;

@Controller
// @Scope("prototype")  设置为多例
public class UserAction {
    @Autowired // Spring 会自动按照类型注入 userService 赋值，我们也可以指定 id 注入 [用得较多]
    @Qualifier("myUserService") // 根据 id 去注入 [用得较少]

    // @Resource(name="myUserService") 使用这种写法可以替换 Autowired Qualifier [用得较少]
    private IUserService userService;

    public void save(User user){
        System.out.println("action 调用 service 保存用户");
        userService.add(user);
    }
}

service 层

package me.yanrs.service;

import me.yanrs.dao.IUserDao;
import me.yanrs.model.User;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.beans.factory.annotation.Qualifier;
import org.springframework.stereotype.Service;

@Service("myUserService")
// @Scope("prototype")  设置为多例
public class UserServiceImpl implements IUserService {
    @Autowired  // Spring 会自动按照类型注入 userDao 赋值 [用得较多]
    @Qualifier("myRepository") // 指定 id 注入 userDao 赋值 [用得较少]

    // @Resource(name="myRepository") 使用这种写法可以替换 Autowired Qualifier [用得较少]
    private IUserDao userDao;

    @Override
    public void add() {
        System.out.println("创建用户");
    }

    @Override
    public void add(User user) {
        System.out.println("service 调用 dao 添加用户");
        userDao.add(user);
    }
}
