Spring Boot RESTful

表现层状态转换：Representational State Transfer，缩写为 REST 是 Roy Thomas Fielding博士于 2000 年在他的博士论 中提出来的一种万维网软件架构风格，目的是便于不同软件/程序在网络（例如互联网）中互相传递信息。表现层状态转换是根基于超文本传输协议(HTTP)之上而确定的一组约束和属性，是一种设计提供万维网络服务的软件构建风格。

PathVariable

使用 PathVariable 注解来获取 URL 中的参数，详细用法如下：

package me.yanrs.springweb.controller;

import me.yanrs.springweb.model.User;
import org.springframework.web.bind.annotation.PathVariable;
import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.web.bind.annotation.RestController;

@RestController
public class RESTfulController {

    // http://127.0.0.1:8080/spring-web/boot/user/1/zhangsan
    // 使用 {} 作为占位符，使用 PathVariable 取得相对应的值
    @RequestMapping("/boot/user/{id}/{name}")
    public Object user(@PathVariable("id") Integer id, @PathVariable("name") String name){
        User user = new User();
        user.setId(id);
        user.setName(name);
        return user;
    }
}

热部署

热部署是指我们修改代码后，服务能自动重启加载新的修改内容。而不是我们每次都需要重新启动。插件为 spring-boot-devtools

<!--Spring Boot 热部署-->
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-devtools</artifactId>
    <optional>true</optional>
</dependency>

在 IDEA 中，每次修改代码后，就算使用了 spring-boot-devtools，项目还是不会自动重启项目的，需要我们 Build 一下。但是 Eclipse 中则不需要 Build，保存就生效了。还有一些问题是，就算使用了 spring-boot-devtools 但是却没有生效，这是就需要手动重新启动项目了。
