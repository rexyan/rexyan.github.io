工作中慢慢要接触 Spring Boot 项目了，但是传统的 Java Web 体系庞大陈杂，现目前使用得比较多的是 Spring Boot， 所以为了让自己有兴趣（尝个甜头）的快速介入，开始尝试 Spring Boot，接触下来发现还可以。

Spring Boot 特性

  • 使用 Spring 项目引导页面可以在几秒构建一个项目
  • 方便对外输出各种形式的服务，如 REST API、WebSocket、Web、Streaming、Tasks
  • 非常简洁的安全策略集成
  • 支持关系数据库和非关系数据库
  • 支持运行期内嵌容器，如 Tomcat、Jetty
  • 强大的开发包，支持热启动
  • 自动管理依赖
  • 自带应用监控
  • 支持各种 IED，如 IntelliJ IDEA 、NetBeans

主入口

SpringWebApplication 是一个带有 main()方法的类，用于启动应用程序

package me.yanrs.springweb;

import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;

@SpringBootApplication
public class SpringWebApplication {
    public static void main(String[] args) {
        // 启动了 Spring boot 程序 （启动 Spring 容器，启动内嵌的 Tomcat）
        SpringApplication.run(SpringWebApplication.class, args);
    }
}

pox.xml

pox.xml 是 Maven 的构建说明文件，这里是因为通过 IDEA 创建的项目，所以会自动加上 Spring boot 开发 web 项目的起步依赖等，如果是直接通过 Maven 创建的，那么需要手动加上。

<?xml version="1.0" encoding="UTF-8"?>
<project xmlns="http://maven.apache.org/POM/4.0.0" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://maven.apache.org/POM/4.0.0 http://maven.apache.org/xsd/maven-4.0.0.xsd">
    <modelVersion>4.0.0</modelVersion>

    <!-- 继承 Spring boot 的父级项目依赖 -->
    <parent>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-parent</artifactId>
        <version>2.1.3.RELEASE</version>
        <relativePath/> <!-- lookup parent from repository -->
    </parent>

    <!--项目描述-->
    <groupId>me.yanrs</groupId>
    <artifactId>spring-web</artifactId>
    <version>1.0.0</version>
    <name>spring-web</name>
    <description>Spring Boot project</description>

    <!--属性配置-->
    <properties>
        <java.version>1.8</java.version>
    </properties>

    <dependencies>
        <!--Spring boot 开发 web 项目的起步依赖 -->
        <dependency>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-starter-web</artifactId>
        </dependency>

        <!--测试的起步依赖-->
        <dependency>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-starter-test</artifactId>
            <scope>test</scope>
        </dependency>

    </dependencies>

    <build>
        <plugins>
            <!--spring boot 提供的项目编译打包插件-->
            <plugin>
                <groupId>org.springframework.boot</groupId>
                <artifactId>spring-boot-maven-plugin</artifactId>
            </plugin>
        </plugins>
    </build>
</project>

运行

新建一个 controller 包，里面新建一个 HelloController 类，浏览器中打开相对应的 /boot/hello 页面就能看到返回值。

package me.yanrs.springweb.controller;

import org.springframework.stereotype.Controller;
import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.web.bind.annotation.ResponseBody;

@Controller // 声明是一个 Controller
public class HelloController {

    @RequestMapping("/boot/hello") // 路由
    public @ResponseBody String hello(){ // ResponseBody 代表返回一个 Json 或者文本
        return "Hello Spring";
    }
}

application.properties

application.properties 是配置整个项目的地方，用 properties 的方式来代替了以前大量 xml 的方式。还可以在 resources 下新建 application.yml 文件。它的作用和 application.properties 是一样的，只不过是以 yml 的方式来进行配置，当二者同时存在时会默认优先使用 properties 文件。我们还可以在 application.properties 中选择不同的环境(若要进行不同环境的区别，需要在 resources 下新建不同的 properties 文件，例如 application-dev.properties，application-pro.properties 等) Spring Boot 会优先使用激活的环境中的配置，如果激活的环境中没有的配置在全局中有，那么就用全局的。当然也支持自定义一些配置。

# 和 yml 文件相比，默认优先使用 properties 文件

# 选择不同环境的配置文件（优先使用激活的环境中的配置，如果激活的环境中没有的配置在全局中有，那么就用全局的）
spring.profiles.active=dev

# 前段页面视图使用 JSP
spring.mvc.view.prefix=/
spring.mvc.view.suffix=.jsp

# 自定义配置
boot.name=第一个 Spring boot 程序
boot.value=Test

获取自定义配置的两种方法

在上面的文件中，我们自定了两组配置，分别为 boot.name 和 boot.value

获取自定义配置-方式1

使用 @Value 注解，里面加上名称，就能直接获取，并赋值给相应对象。访问 /boot/config 看到具体的效果。

package me.yanrs.springweb.controller;

import me.yanrs.springweb.config.ConfigInfo;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.beans.factory.annotation.Value;
import org.springframework.stereotype.Controller;
import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.web.bind.annotation.ResponseBody;

@Controller
public class ConfigInfoController {

    // 读取配置文件中的默认配置, 方式1(使用 @Value 注解)
    @Value("${boot.name}")
    private String name;

    @Value("${boot.value}")
    private String value;

    @RequestMapping("/boot/config")
    public @ResponseBody String configInfo(){
        return name + "----" + value;
    }
}
获取自定义配置-方式2

使用 ConfigurationProperties 将其配置成一个对象

书写一个类，使用 ConfigurationProperties 注解，并加上前缀，并创建相对应的属性，赋予 get，set 方法。这样就能获取到相对应前缀的自定义对象了。加上 Component 让其成为一个组件，在使用的地方直接使用 Autowired 进行自动载入。

package me.yanrs.springweb.config;

import org.springframework.boot.context.properties.ConfigurationProperties;
import org.springframework.stereotype.Component;

@Component // 表示是一个组件
@ConfigurationProperties(prefix = "boot") // 获取自定义对象，前缀是 boot
public class ConfigInfo {
    private String name;
    private String value;

    public String getName() {
        return name;
    }

    public void setName(String name) {
        this.name = name;
    }

    public String getValue() {
        return value;
    }

    public void setValue(String value) {
        this.value = value;
    }
}

访问 /boot/config 就能看到具体效果

package me.yanrs.springweb.controller;

import me.yanrs.springweb.config.ConfigInfo;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.beans.factory.annotation.Value;
import org.springframework.stereotype.Controller;
import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.web.bind.annotation.ResponseBody;

@Controller
public class ConfigInfoController {

    // 读取配置文件中的默认配置, 方式1(使用 @Value 注解)
    @Value("${boot.name}")
    private String name;

    @Value("${boot.value}")
    private String value;

    // 读取配置文件中的默认配置, 方式2（使用 ConfigurationProperties 将其配置成一个对象）
    @Autowired
    private ConfigInfo configInfo;

    @RequestMapping("/boot/config")
    public @ResponseBody String configInfo(){
        return name + "----" + value + "----" + configInfo.getName() + "----" + configInfo.getValue();
    }
}

集成 Spring MVC

RestController

RestController = @Controller + @ResponseBody。是 Spring4 后新增的注解，功能是用于返回字符串或 json 数据

GetMapping

@GetMapping = RequestMethod.GET 请求 + @RequestMapping

PostMapping

@PostMapping = RequestMethod.POST 请求 + @RequestMapping

package me.yanrs.springweb.controller;

import me.yanrs.springweb.model.User;
import org.springframework.web.bind.annotation.*;

// RestController = @Controller + @ResponseBody。是 Spring4 后新增的注解，功能是用于返回字符串或 json 数据
@RestController
public class MVCController {

    // value 是 URL 的地址，method 是该地址的请求方法
    @RequestMapping(value = "/boot/getUser", method = RequestMethod.GET)
    public Object getUser(){
        User user = new User();
        user.setId(1);
        user.setName("汪汪");
        return user;
    }

    // @GetMapping = RequestMethod.GET 请求 + @RequestMapping。和上面的功能等价。
    @GetMapping("/boot/getUser1")
    public Object getUser1(){
        User user = new User();
        user.setId(1);
        user.setName("汪汪");
        return user;
    }

    // @PostMapping = RequestMethod.POST 请求 + @RequestMapping
    @PostMapping("/boot/getUser2")
    public Object getUser2(){
        User user = new User();
        user.setId(1);
        user.setName("汪汪");
        return user;
    }

    // 同理，还有 PutMapping，DeleteMapping
}

书写一个 Controller 返回 jsp 页面, 访问 /boot/index 就可以看到 msg 信息了。

package me.yanrs.springweb.controller;

import org.springframework.stereotype.Controller;
import org.springframework.ui.Model;
import org.springframework.web.bind.annotation.GetMapping;

@Controller
public class JSPController {
    @GetMapping("/boot/index")
    public String index(Model model){
        model.addAttribute("msg", "spring boot 集成 jsp");
        return "index";
    }
}
